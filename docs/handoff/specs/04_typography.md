# Русские типографские правила для Самосайта

> **Контекст:** обязательные правила для всех публичных текстов — лендинг, customer-сайты, customer admin, email-уведомления, push, intake-формы.
> Эти правила применяются **до** деплоя на прод, через скрипт или ручную проверку.

---

## 1. Без точек в конце ёмких фраз

**ЗАПРЕЩЕНО** ставить точку в конце:
- H1 / H2 / H3 / H4 / H5 / H6 — заголовков всех уровней
- Section subtitle (под H2)
- Body-параграфов из **одного** предложения
- Caption / меток / мини-описаний (mono uppercase)
- Текстов CTA-кнопок
- Микрокопии под кнопками
- Buллетов в списках, если буллет — самостоятельное предложение

**Можно ставить точку:**
- В FAQ-ответах (полные предложения в диалоговой форме)
- В testimonial-цитатах (живая речь)
- В body-параграфах из 2+ предложений — между ними нужны, в последнем — **по решению** (для маркетинга — без)
- В legal-текстах (privacy / offer / cookie)

### Примеры

```
ПЛОХО (с точкой):
H2: «Восемь «сам». Поэтому он Самосайт.»

ХОРОШО (без точек, em-dash вместо точки):
H2: «Восемь «сам» — поэтому он Самосайт»
```

```
ПЛОХО:
sub: «Подойдёт любая ссылка, где про вас уже что-то написано или показано.»

ХОРОШО:
sub: «Подойдёт любая ссылка, где про вас уже что-то написано или показано»
```

## 2. Non-breaking space (`\u00A0`) после коротких слов

Чтобы короткие предлоги/союзы не висели в конце строки, после них **обязательно** ставить `\u00A0` (nbsp, U+00A0) перед следующим словом.

### Список коротких слов

**1-буквенные предлоги/союзы:**
```
в, с, к, у, о, и, а
```

**2-3-буквенные предлоги/союзы (на которые часто переносится):**
```
на, за, от, до, по, из, для, что, как, или,
но, же, ли, то, не, без, со, об, ко, во, при
```

### Примеры

```
ПЛОХО:  «Самосайт собирает сайт из
         того, что у вас уже есть»
                          ^^^         ← «у» висит в конце

ХОРОШО: «Самосайт собирает сайт из\u00A0
         того, что у\u00A0вас уже есть»
                  ^^^^^^^             ← «у вас» вместе
```

```
ПЛОХО:  «Сделано на
         Самосайте»

ХОРОШО: «Сделано на\u00A0Самосайте»
```

```
ПЛОХО:  «Для записи в
         Telegram»

ХОРОШО: «Для\u00A0записи в\u00A0Telegram»
                            ^^^^^^^^^^^^^
```

## 3. Тире vs дефис

| Символ | Unicode | Назначение | Пример |
|---|---|---|---|
| `—` | U+2014 (em-dash) | Смысловая пауза | «От вас — одно действие» |
| `–` | U+2013 (en-dash) | Числовые диапазоны | «3–5 дней», «10:00–20:00» |
| `-` | ASCII hyphen | Составные слова | «бизнес-карточка», «1 200+» |

**Em-dash** окружается обычными пробелами (`текст — текст`), на тонком экране это даёт явный визуальный paragraph-break.

**НЕЛЬЗЯ:** ставить пробелы вокруг en-dash или hyphen.

## 4. Кавычки

| Уровень | Символ | Пример |
|---|---|---|
| Основные | `«…»` (ёлочки) | «Запишитесь онлайн» |
| Вложенные | `„…"` (лапки) | «Мастер сказал: „Завтра в 14:00"» |

**Запрещено** в публичной копи:
- Прямые `"…"` или `'…'` (ASCII)
- Тильда `~…~`
- Тех. кавычки `\` / `/`

---

## 5. Mono numbers

В моно-шрифте (JetBrains Mono) числа отделяются **обычным** пробелом, не nbsp.

```
ХОРОШО: 1 200+    1 284    990 ₽    +7 999 111-11-11
        ↑ regular space (U+0020)

ПЛОХО:  1&nbsp;200+         ← ломает Mono-семейство
        1\u00A0200          ← то же
```

**Исключение:** в цене `990 ₽` — пробел между числом и `₽` можно делать `&nbsp;` чтобы не разорвать визуально, **но**:
- В Mono — обычный пробел
- В sans-serif (заголовок Pricing) — `&nbsp;`

---

## 6. Спец. символы

| Символ | Unicode | Назначение |
|---|---|---|
| `·` | U+00B7 (middle dot) | Разделитель в meta-строках: «Записаться · 30 сек», «Я.Карты · 12 апр» |
| `…` | U+2026 (ellipsis) | Многоточие (одним символом, не три точки) |
| `№` | U+2116 | Номер (для лицензий, документов, etc.) |
| `™` | U+2122 | Не используем |
| `°` | U+00B0 | Градусы (если упоминаются температуры) |
| `±` | U+00B1 | Не используем в маркетинге |

## 7. Регистр

- **CAPS UPPERCASE** только для:
  - Mono-eyebrows (если используются): `★ ЛУЧШИЙ`, `СКОРО ПОДКЛЮЧИМ`
  - Чек-сертификаты, ID, technical artifacts
  - Watermark на customer-сайте: `СДЕЛАНО НА САМОСАЙТЕ`
- **Title Case** не используем (это англоязычный паттерн).
- **Стандартный** регистр для H1/H2/H3/body — первая буква заглавная, остальное прописные. Бренд «Самосайт» — всегда с заглавной.

## 8. Бренд «Самосайт»

| Падеж | Форма | Пример |
|---|---|---|
| Им. | Самосайт | «Самосайт собирает сайт за пару минут» |
| Род. | Самосайта | «бот Самосайта», «возможности Самосайта» |
| Дат. | Самосайту | «к Самосайту подключаются за минуту» |
| Вин. | Самосайт | «собрать мой Самосайт» |
| Тв. | Самосайтом | «работает с Самосайтом» |
| Пр. | Самосайте | «сделано на Самосайте» |

**Всегда** кириллицей, **всегда** с заглавной. Латинское `Samosite` — только в коде, в `:host` названиях, в `samosite.online` URL.

## 9. Автоматизация

### 9.1 Скрипт обработки JSX/TSX

```js
// transform-russian-typography.js
const SHORTS = ['в','с','к','у','о','и','а','на','за','от','до','по','из','для','что','как','или','но','же','ли','то','не','без','со','об','ко','во','при'];

function applyTypography(source) {
  const lines = source.split('\n');
  return lines.map(line => {
    if (!/[А-Яа-яЁё]/.test(line)) return line;
    let result = line;
    for (const w of SHORTS) {
      // " <short> " → " <short>\u00A0"
      const re = new RegExp(`(\\s|>|«|\\()${w} (?=[А-Яа-яЁё«])`, 'g');
      result = result.replace(re, `$1${w}\u00A0`);
    }
    return result;
  }).join('\n');
}
```

Этот скрипт уже применён ко всем JSX-файлам канона. Для прод-кода нужно:

1. Либо **копировать строки из канона побайтово** (с уже встроенными nbsp)
2. Либо **прогнать скрипт** через все `.tsx` / `.j2` файлы в `/code/` перед билдом
3. Либо использовать **npm-пакет** типа `russian-typograph` или `tg-typograph` для on-the-fly обработки

### 9.2 ESLint-rule (рекомендуется)

## 10. Чек-лист для ревью

При проверке любого нового текста / правки в копи:

- [ ] Нет точек в конце H1/H2/H3/sub/body-параграфов из 1 предложения
- [ ] После «в», «с», «к», «у», «о», «и», «а», «на», «за», «от», «до», «по», «из», «для», «что», «как», «или», «но», «же», «ли», «то», «не», «без» — стоит `\u00A0` перед следующим словом
- [ ] Тире `—` для пауз, дефис `-` для составных слов
- [ ] Кавычки `«…»` основные, `„…"` вложенные (не ASCII)
- [ ] Mono numbers через regular space
- [ ] CAPS только для eyebrow-mono и watermark
- [ ] «Самосайт» с заглавной, в правильном падеже
