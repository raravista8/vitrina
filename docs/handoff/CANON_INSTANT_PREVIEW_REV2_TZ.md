# ТЗ-правки rev.2 к instant-preview — вход «ниша-демо», поиск по названию, морф

> **Адресат:** команда Claude Design (авторы канона `@samosite/canon`).
> **Заказчик:** vitrina / Самосайт прод.
> **Тип:** правки поверх УЖЕ реализованного instant-preview (`CANON_INSTANT_PREVIEW_TZ.md`, rev.1). Сделанное в rev.1 — `S3_StepBuilding`, `S3_StepPreview` (rich/sparse), `PreviewDraft`, состояния failed/timeout, контакт-копи — **не перерисовывается**; rev.2 добавляет вход ДО ссылки и меняет шаг источника.
> **Суть одним абзацем:** wow в rev.1 всё ещё заперт за ссылкой, а 20 из 24 открывших модалку приходят с пустыми руками. rev.2 переворачивает вход: тап по нише → мгновенный пример сайта (0 секунд, без сети) → «заменить на ваши данные» → источник ищется по названию (ссылка не нужна) → пример морфится в его черновик. Принцип: **показывать раньше, чем просить**.

## 1. Карта флоу rev.2 — целиком

```
Лендинг, клик CTA
│
├─ Hero-инпут ПУСТ (большинство, 20/24) ──► Шаг 0 «Ниша»
│                                             │ тап по нише
│                                             ▼
│                                           Шаг 0b «Пример» (0 c, штамп ПРИМЕР)
│                                             │ «Заменить на ваши данные»
│                                             ▼
│                                           Шаг 1 «Источник»
│                                             ├─ A: поиск по названию (приоритет)
│                                             ├─ B: вставить ссылку
│                                             └─ C: фото → photo-ветка (без изменений)
│                                             │ источник определён
│                                             ▼
├─ Hero-инпут с ok-ссылкой ────────────────► Шаг 2 «Сборка» (rev.1; + морф, §5)
│   (Шаг 0/0b/1 пропускаются)                 ▼
│                                           Шаг 3 «Превью» (rev.1 rich/sparse)
├─ Hero-инпут с soon-ссылкой ──► waitlist     ▼
│   (как сейчас, без изменений)             Шаг 4 «Контакт» (rev.1 копи)
└─ клик «фото» ──► photo-ветка                ▼
    (как сейчас, без изменений)             Шаг 5 «Готово» (rev.1)
```

Прогресс-доты: **4** — Источник · Превью · Контакт · Готово. Шаги 0/0b дотов не имеют (это витрина, не воронка); доты появляются с шага 1. Сборка = loading-состояние дота «Превью».

Кнопка «← Назад» работает на каждом шаге: 0b→0, 1→0b, превью→1 («Изменить источник»), контакт→превью. Закрытие модалки с любого шага — текущий крестик-паттерн.

## 4. Лейаут-логика по вьюпортам

- **Desktop (модалка ≥720):** шаги 0/1 — центрированная колонка ~560; шаг 0b и превью — широкий режим: рамка сайта слева (~62 %), панель (штамп-строка, свотчи, CTA, микрокопи) справа.
- **Mobile (фуллскрин, 390/375):** шаги 0/1 — одна колонка; шаг 0b и превью — сайт на весь экран, **закреплённая нижняя панель** (высота ≤96): строка-пояснение петитом, свотчи и CTA в один ряд. Грид плашек шага 0 — 2 колонки, плашка ≥44 px высотой. Всё обязано выживать на 375.
- Скролл сайта внутри рамки не конфликтует со скроллом модалки (рамка — отдельная scroll-область).

## 5. Морф «пример → черновик» — спецификация по слотам

Главный craft-момент. Тот же артборд: компонент превью НЕ размонтируется между 0b → сборка → превью, меняются только данные. Никаких «закрыли пример — показали спиннер — открыли превью».

| Слот | Когда меняется | Как |
|---|---|---|
| Штамп | старт сборки | `ПРИМЕР` → `СОБИРАЕМ…` → `ЧЕРНОВИК` |
| Название + категория | первый ответ poll (`stage: fetching` пройден) | кросс-фейд текста |
| Галерея / photoSrc | по мере `counts.photos` | фото примера заменяются его фото по одному, слева направо, кросс-фейд ~200 мс; лишние плейсхолдеры скрываются |
| Отзывы (quote) | `stage: reviews` | кросс-фейд цитаты |
| Услуги/цены | `draft.services` непустой | строки появляются стэггером; пусто → блок примера скрывается |
| Рейтинг/район (stats) | вместе с draft | кросс-фейд |
| Тема | только если меняется (§2, правило `userThemeTouched`) | одним переходом ДО подмены контента, не вместе |

Этапы-строки сборки на десктопе — в правой панели; на мобиле — компакт-строка над нижней панелью. Easing — спокойный, без bounce; reduced-motion → мгновенные подмены без анимации.

## 6. Controlled API — дельты (паттерн 0.3.0/0.9.1: zero-prop = canvas-mock)

```ts
// НОВОЕ — словарь ниш, единый для канваса и прода (аналог SOURCE_LIB)
export interface NicheItem {
  id: string;                 // 'manicure' | 'brows' | …
  label: string;              // «Маникюр»
  synonyms: string[];         // ['ногти', 'гель-лак', …] — матчинг свободного поля
  theme_id: string;           // тема по умолчанию
  family_id: string;
  theme_options: string[];    // 2–3 theme_id для свотчей
}
export const NICHE_LIB: NicheItem[];   // 10 ниш из §3.1

// НОВОЕ
S3_StepNiche({
  niches?: NicheItem[],            // default NICHE_LIB
  freeText, onFreeTextChange,
  onPick: (nicheId: string) => void,
  onShowExample: (freeText: string) => void,
})

// НОВОЕ
export interface SourceCandidate {
  id: string;                      // opaque, уходит в POST /api/preview/draft
  name: string;
  address: string;
  rating: { value: number; count: number } | null;
  photo: string | null;
}
S3_StepSource({
  mode: 'search' | 'link',
  query, city, onQueryChange, onCityChange,
  searching: boolean,
  candidates: SourceCandidate[] | null,    // null = ещё не искали
  searchError: 'none' | 'empty' | 'network' | 'ratelimited',
  retryAfterSeconds?: number,
  onSearch, onPickCandidate: (id) => void, onNotMine,
  // mode='link' — текущие пропсы линк-инпута + бейджа rev.1 как есть
  onSwitchMode: (m) => void,
  onPhotoBranch,
})

// ПРАВКИ существующих rev.1
S3_StepBuilding({ …rev.1, baseDraft?: PreviewDraft })
//  baseDraft есть → морф-режим §5 (рендер chassis с примером, подмена слотов);
//  нет → standalone-экран сборки rev.1 (путь hero-URL)
S3_StepPreview({ …rev.1, variant: 'demo' | 'rich' | 'sparse' })
//  'demo': штамп ПРИМЕР, панель «Что мы нашли» скрыта,
//  CTA «Заменить на ваши данные», микрокопи «Меньше минуты…»
```

Демо-фикстуры ниш = существующие `samplePresets` (маппинг в `NICHE_LIB`). Никаких сетевых вызовов внутри канона. TS-типы (`NicheItem`, `SourceCandidate`) — экспортировать.

## 8. Селекторы и аналитика

Каждый шаг несёт `data-intake-step="niche | demo | source | building | preview | contact | confirm"`; плашки — `data-niche-id`; карточки кандидатов — `data-candidate-idx`; CTA шага 0b — `data-cta="claim-demo"`, превью — `data-cta="claim-draft"`. Это хуки для целей Метрики (замер новой воронки: ниша → демо → источник → превью → заявка) — не менять без согласования.

## 9. Edge cases

- **Ниша выбрана, источник говорит другое** (выбрал «Маникюр», ссылка — барбершоп): данные источника выигрывают, категория обновляется при морфе; тема — по правилу `userThemeTouched` (§2).
- **Свободное поле без матча** → generic-тема, подпись категории = введённый текст.
- **Кандидат без фото** → после выбора обычный путь, скорее всего `preview-sparse`.
- **Город пуст** → поиск без города; кандидаты различаются адресами.
- **Повторное открытие модалки** в той же сессии → монтирование на сохранённом шаге (state у консьюмера, канон уже поддерживает с 0.3.0).
