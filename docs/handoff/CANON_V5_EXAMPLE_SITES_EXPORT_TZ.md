## 2. Правила конвертации canvas → пакет (грабли прошлых релизов)

Каждый пункт — реальный инцидент; проверяйте перед упаковкой:

1. **Никаких следов канваса:** ни IIFE-обёрток (`if (!window.__…) {…`), ни
   голых баннер-строк, ни `window.*`-глобалов, ни `dcExport` (утёкший
   IIFE-opener сломал нам SSR в 0.8.0).
2. **Никаких self-ref констант** вида `const X = X;` — конвертер 0.8.0
   превратил `window.PresetRenderer` в TDZ-краш.
3. **Никаких дублей экспортов:** инлайн `export` + нижний `export {}`-блок
   с теми же именами валит ESM-сборку («Multiple exports», инцидент
   `in2.tsx` в 0.12.0). Один способ экспорта на файл.
4. **`'use client'`** первой строкой обоих новых файлов.
5. **Без `React.lazy`/`import()`:** «lazy, load-order-safe» из прототипа —
   это канвас-трюк порядка `<script>`; в пакете `FullComp` — обычные
   статические импорты (tsup-чанки + `loading="lazy"` картинок достаточно;
   маунт только открытого примера уже даёт отложенный рендер).
6. **Свежесть файлов в зипе:** перед отправкой сверьте, что зип несёт
   именно новые файлы — в 0.9.1 рекурсивное копирование затёрло свежий
   `customer/index.tsx` старым прямо в архиве.
7. **Чистота типов:** пакет собирается нашим tsup с `dts: false`, но код
   должен парситься как TS (у нас строгий ESLint на консьюмере ambient-шима
   нет только у типов, не у синтаксиса).

## 3. База для `v5.tsx` — наш вендоренный файл, не ваш исходник

Ваш исходник `v5.tsx` **разошёлся с продом**: у нас консьюмер-патч
`[vitrina] copy patch — V5_Hero lead` (SEO-лид героя, Known gaps в
CHANGELOG), которого у вас может не быть (прецедент 0.13.0 — тогда зип вёз
0.12.0-состояние, и мы файл не копировали вовсе).

Требование к поставке `v5.tsx`:

- собирайте изменения **поверх приложенного нами файла** (передадим наш
  вендоренный `packages/canon/src/landing/v5.tsx` вместе с этим ТЗ);
- в зип положите **и полный файл, и unified-diff** (`v5.tsx.diff` против
  нашей базы) — по диффу мы за минуту убеждаемся, что затронут только
  блок Examples/SiteViewer;
- если диффом всё-таки зацепите что-то вне блока — это блокер поставки,
  а не сноска в CHANGELOG.

## 4. CHANGELOG 0.14.0 — обязательный состав секции

1. Что нового: 5 полноразмерных сайтов, takeover-просмотрщик, controlled-API
   (`viewerId`/`onViewerChange`, `FullComp?`), фикс scroll-lock
   (`body.is-locked` впервые получил CSS-правило — отметить как fix).
2. **Реестр копи** — недоделка приёмки основного ТЗ §9, теперь обязателен:
   таблица всех новых строк (прайсы, отзывы с атрибуцией, «о мастере»,
   футеры, лейблы навигации) по 5 сайтам. Дальше копи-правки идут только
   через реестр. Типографика — по `specs/04_typography.md` (nbsp, без точек
   в заголовках, «ёлочки»).
3. Если меняли `note`-строки карточек карусели (вариант из основного ТЗ §2
   про легенды brows/barber) — старые/новые строки таблицей.
4. **Ассеты:** полный список Unsplash-id новых фото + локальные файлы
   (`img/nails.png` и что добавилось) — по образцу секции ассетов 0.12.0.
5. data-* хуки: `data-example-site` / `data-example-open` /
   `data-example-back`; `data-entry="example-<id>"` — без изменений.
6. Migration-заметка консьюмеру: как включить controlled-режим; поведение
   zero-prop (открытие теперь ведёт на полноразмерник — это не регресс,
   это фича).
7. Список byte-identical модулей (§1).
