# ТЗ — мгновенный превью сайта в intake («сайт до контакта»)

> **Адресат:** команда Claude Design (авторы канона `@samosite/canon`).
> **Заказчик:** vitrina / Самосайт прод.
> **Тип:** новая фича в intake-флоу — самое важное продуктовое изменение с запуска. Не рестайл.
> **Желаемый артефакт:** canon 0.10.x — новые шаги SubmitModal (controlled API, как 0.3.0/0.9.1) + canvas-артборды всех состояний.
> **Канал ответа:** как обычно — zip `canon X.Y.Z` + CHANGELOG. Вопросы по контракту `PreviewDraft` — приоритетно, бэкенд строится параллельно по этому же документу (§6 — замороженный контракт).

## 1. Зачем — данные, а не вкус

Прогнали тестовую кампанию в Я.Директе (поиск). Воронка на реальном холодном трафике:

| Шаг | Человек | Конверсия от кликов |
|---|---|---|
| Клики по объявлению (CTR 4,84 % — объявления ок) | 89 | 100 % |
| Открыли SubmitModal | 24 | 27 % |
| **Вставили ссылку (шаг 1)** | **4** | **4,5 %** |
| Дошли до контактного шага | 5 | 5,6 % |
| Отправили заявку | 2 | 2,25 % |
| Ответили на связь после заявки | 0 | 0 % |

Два слома:

1. **24 → 4 внутри модалки.** Человек кликнул «Собрать сайт за 2 часа» — ему интересно. Но первый же экран просит «дай ссылку на свой Telegram / Я.Карты» — то есть доверие ДО того, как показана ценность. При этом карусель примеров — самая вовлекающая секция лендинга (13 % целевых визитов «карусель смотрел», лучший показатель страницы): **люди хотят увидеть результат, мы показываем форму.**
2. **2 → 0 после заявки.** Заявка = «оставь контакт и жди». Импульс умирает за минуты; когда founder выходит на связь через часы — человеку уже неинтересно. Ему не за чем возвращаться: он ничего «своего» не видел.

## 2. Продуктовая идея

Между «вставил ссылку» и «оставь контакт» появляется **живой черновик его сайта**, собранный из его собственных данных за 15–40 секунд. Контакт перестаёт быть входным билетом и становится способом **забрать уже увиденный сайт**.

Это чинит оба слома: появляется причина вставить ссылку («покажем, как будет выглядеть ваш сайт — бесплатно, без контакта»), и появляется владение («это МОЙ сайт с МОИМИ фото») до контактного шага — за этим возвращаются.

Канон уже почти всё для этого имеет — §3.

## 3. Что переиспользуем из канона (НЕ рисовать заново)

- **`@samosite/canon/presets`** (с 0.7.1): `PresetRenderer`, `MiniChrome`, 5 layout-семейств × 16 тем, слоты `SlotContent` (name, category, services/menu, stats, quote, gallery, photoSrc). **Превью = `PresetRenderer`, в который вместо фикстур подставлен `PreviewDraft` (§6).** Новый рендерер сайтов не нужен.
- **`SubmitModal`** (0.3.0 controlled API) — расширяется новыми шагами в link-ветке; photo-ветка не меняется (§9).
- **`SourceDetectionBadge`** / зелёная плашка «Распознали: …» на шаге 1 — остаётся, это вход в сборку.
- Токены, типографика, `Btn`/`Spinner`/примитивы — без изменений.

## 4. Новый флоу (link-ветка)

```
Сейчас:  Шаг 1 (ссылка) → Шаг 2 (контакт) → Шаг 3 (подтверждение)

Станет:  Шаг 1 (ссылка, как сейчас)
         → Шаг 2 НОВЫЙ «Сборка» (15–40 c, живой прогресс)
         → Шаг 3 НОВЫЙ «Превью» (черновик сайта + что нашли)
         → Шаг 4 (контакт — текущий S3_StepContact, новый заголовок-копи)
         → Шаг 5 (подтверждение — текущий S3_FinalConfirm)
```

Прогресс-доты модалки: 3 → 4 видимых шага в link-ветке (сборка и превью — один дот «Превью», сборка — его loading-состояние; решение за вами, но дотов не больше 4).

### Шаг «Сборка» — требования

- **Реальный прогресс, не фейковый спиннер.** Этапы привязаны к настоящим событиям парсинга (бэкенд отдаёт их в poll-ответе, §6): «Читаем источник» → «Забрали N фото» → «Нашли M отзывов» → «Подбираем стиль под нишу». Числа — настоящие, по мере появления.
- Скелет превью проявляется по мере данных (карточка `MiniChrome` уже на экране, блоки заполняются) — НЕ пустой экран с прогресс-баром.
- Анти-цель: «AI-театр» длиннее реального парсинга. Если бэкенд собрал за 8 секунд — показываем за 8 секунд.

### Шаг «Превью» — требования

- **Герой экрана — сам сайт**: `PresetRenderer` в `MiniChrome`, скроллится внутри рамки. Тема выбрана автоматически по нише (маппинг ниша → тема уже есть в preset-фикстурах).
- Лейбл-штамп «ЧЕРНОВИК» + строка «Полную версию соберём за 2 часа» — мы показываем эскиз, не финал. Без обещания «это готовый сайт».
- Панель «Что мы нашли»: `N фото · M отзывов · услуги · адрес/район` — mono-петит, язык документа (как бейджи intake).
- Переключатель темы: **максимум 2–3 пресета** (prev/next или 3 свотча). Не 16. Выбор уходит в payload заявки.
- CTA: **«Забрать сайт — бесплатно»** → шаг контакта. Вторичная ссылка «Изменить источник» → назад на шаг 1.
- Превью доступен ДО контакта осознанно (пусть скриншотят — это маркетинг). Забрать/опубликовать — только через контакт.

### Шаг «Контакт» — только копи

Компонент тот же (`S3_StepContact`). Заголовок меняется с «Куда вам писать?» на смысл «Куда прислать готовый сайт?» — sub: «Соберём полную версию и пришлём ссылку — обычно в течение 2 часов». Финальные строки — по `docs/COPY.md` (без тире, CTA = результат).

## 5. Состояния (все нужны на canvas)

| Состояние | Когда | Что показываем |
|---|---|---|
| `building` | 0–40 c | Этапы + живые счётчики + проявляющийся скелет |
| `preview-rich` | есть фото (≥3) и/или отзывы | Полный превью |
| `preview-sparse` | распарсили только имя/категорию/адрес | Превью на шаблонных фото-плейсхолдерах ниши + явная строка «Фото добавим из источника при полной сборке»; CTA тот же |
| `preview-failed` | парсер упал / источник пустой | Мягкий fallback: «Не дотянулись до источника — соберём вручную за 2 часа» → сразу контактный шаг. Текущий флоу, ничего не ломается |
| `timeout` | > 40 c | «Собираем дольше обычного. Оставьте контакт — пришлём готовый черновик» → контактный шаг (сборка продолжается асинхронно) |
| `soon`-источник | vk/instagram/… | Как сейчас (waitlist-ветка), превью не запускается |

Viewports: desktop 1440 + mobile 390/375 (модалка фуллскрин на мобиле — превью в ней должен выживать на 375).

## 6. Контракт `PreviewDraft` — замороженный, бэкенд строится по нему

Что бэкенд реально достаёт за ≤ 30 секунд **без LLM в синхронном пути** (LLM-тексты приходят позже, в полной сборке — в превью тексты шаблонные из niche-словаря + сырые куски источника):

| Источник | name | category | photos | reviews | services | address/район | rating |
|---|---|---|---|---|---|---|---|
| `yandex_maps` (Geosearch) | ✓ | ✓ | частично | ✓ (count+тексты) | частично | ✓ | ✓ |
| `telegram` (t.me/s/) | ✓ (title) | — (niche guess) | ✓ (посл. посты, до 9) | — | — | — | — |
| `website` (og-теги) | ✓ | — | og:image (1) | — | — | — | — |
| `twogis`, `avito` | ✓ | ✓ | не гарантируем | частично | частично | ✓ | частично |

```ts
export interface PreviewDraft {
  source: 'yandex_maps' | 'telegram' | 'twogis' | 'avito' | 'website';
  name: string;                      // всегда есть (иначе preview-failed)
  category: string | null;           // ниша; null → generic-тема
  district: string | null;           // «Петроградский район» / null
  rating: { value: number; count: number } | null;
  photos: string[];                  // URL, 0..9; <3 → preview-sparse
  reviews: { author: string; text: string; rating: number }[]; // 0..4, сырые
  services: { title: string; price: string | null }[];         // 0..6
  theme_id: string;                  // авто-подбор бэкендом, из 16 существующих
  family_id: string;                 // одно из 5 семейств
}
```

Состояния сборки (poll-ответ): `{ status: 'building' | 'ready' | 'failed', stage: 'fetching' | 'photos' | 'reviews' | 'styling', counts: { photos: number, reviews: number }, draft?: PreviewDraft }`.

Если по макетам понадобится поле сверх этого — это **запрос на изменение контракта**, согласуем явно (дешёвое «давайте ещё свежесть/часы работы» может не парситься за 30 c).

## 7. Controlled API (требование, как 0.3.0 / 0.9.1)

Новые экспорты из `@samosite/canon/intake` — **сразу controlled**, zero-prop = canvas-mock (back-compat паттерн 0.9.1):

```ts
S3_StepBuilding({ stage, counts, draftSkeleton?: Partial<PreviewDraft> })
S3_StepPreview({
  draft: PreviewDraft,
  themeOptions: string[],        // 2-3 theme_id на переключатель
  activeTheme: string,
  onThemeChange: (id: string) => void,
  onClaim: () => void,           // CTA «Забрать сайт — бесплатно»
  onBack: () => void,            // «Изменить источник»
  variant?: 'rich' | 'sparse',
})
```

- TS-типы `PreviewDraft`, `BuildStage` — экспортировать.
- Никаких сетевых вызовов внутри канона — данные и колбэки только через props (сеть на консьюмере, как everywhere).
- `SubmitModal` step-машина расширяется; существующие пропсы 0.3.0 не ломать (additive). Photo-ветка проходит мимо новых шагов без изменений.
- Рендер превью — через существующий `PresetRenderer` с маппингом `PreviewDraft → SlotContent` (маппер можно положить в canon, он чистый).

## 8. Тон и стоп-лист

- Копи — по `docs/COPY.md` + `specs/04_typography.md`: без точек в коротких фразах, «Самосайт» кириллицей, CTA = результат («Забрать сайт — бесплатно», не «Продолжить»).
- Без промо-вайба, без «−XX%», без конфетти на появление превью. Появление черновика — спокойный, уверенный момент «вот ваш сайт», не аттракцион.
- Язык документа на панели «Что мы нашли»: моноцифры, капс-лейблы — как в бейджах источника.

## 9. Чего НЕ делаем в 0.10.x (явный out of scope)

- Превью в **photo-ветке** (vision-пайплайн не укладывается в 30 c) — photo-флоу как есть.
- Выбор из всех 16 тем, кастомизация цветов/шрифтов на превью.
- LLM-тексты в превью (приходят в полной сборке).
- Сохранение черновика в аккаунт / шаринг черновика по ссылке.
- Изменения лендинга вне SubmitModal (Hero-копи под превью обновим консьюмер-сайд отдельно).
