## 2. Принцип упаковки: новый namespace, старое не трогать

Прод сейчас потребляет intake-экспорты 0.3.0/0.10/0.11 (`SubmitModal`, `S3_*`). Новый флоу — **параллельный набор компонентов**, а не мутация старых:

- Новые экспорты из `@samosite/canon/intake` с префиксом **`In2_`** (или предложите свой — но единый и новый): старые `S3_*`/`SubmitModal` остаются byte-identical, прод переключается атомарно на нашей стороне.
- Релиз получается **MINOR, additive**. Депрекейт старого флоу пометим в CHANGELOG, удаление — отдельным мажором позже.
- Внутри новых компонентов **нет** fetch / `Date.now()` / `Math.random()` / localStorage — данные пропсами, действия колбэками, персист черновика делает консьюмер. Zero-prop = canvas-mock (обязательный паттерн).

## 3. Декомпозиция и controlled-контракт (по шагам спеки)

Ниже — минимально необходимые данные/колбэки на каждый экран. Точные имена пропсов — на ваше усмотрение, но покрытие состояний и экспорт TS-типов (`In2_*Props`) обязательны.

### 3.0 `In2_Modal` — оболочка и роутер

- Пропсы: `step` (01|source|recognize|confirm|booking|contacts|done), `onStepChange`, `onClose`, `closeConfirm` (показ «✕?»-подтверждения — управляется консьюмером), `restoredBanner` (баннер «↺ черновик восстановлен» + `onDraftReset`).
- Окно 960px на всех шагах (десктоп), размер не скачет; двухколоночная сетка «слева путь / справа контекст» — внутри шагов.
- Ошибка отправки «⚠»: проп `submitError` + CTA «Повторить отправку» (`onRetry`) — форма не размонтируется.

### 3.1 `In2_StepExample` — шаг 01 «Пример»

- Табы ниш: Маникюр · Брови и ресницы · Барбершоп · Косметолог · Колорист · Другое. Пропсы: `niche`, `onNicheChange`, `onClaim` (CTA «Собрать такой для меня →»), `loading` (скелет 01·з; CTA активна и при скелете — по спеке).
- **Пример = полный лендинг ниши со скроллом внутри окна**: фото-герой с рейтингом, прайс 5 позиций с пояснениями, CTA «Записаться», 3 отзыва с именами и источниками, галерея 2×2, финальный CTA, футер с адресом, address-bar вида `anna-nails.samosite.ru`.
- Рендерить через существующий **`PresetRenderer`** (`@samosite/canon/presets`) + новые фикстуры: **`NICHE_LIB_V2`** — 5 полных лендингов (свои фото, палитра от фото героя, тексты, рыночный прайс, отзывы) + заглушка «Другое» («Соберём под ваше направление»). Не собирать отдельный рендерер сайтов.
- Экспортировать `NICHE_LIB_V2` (id, label, фикстура) — консьюмер шлёт `demo_shown`/`niche_selected` и кладёт нишу в заявку.

### 3.2 `In2_StepSource` — шаг 02 «Источник», 4 равноправных пути

- Пропсы: `path` (name|screenshot|link|photo), `onPathChange` — **переключение не стирает введённое** (стейт всех путей живёт у консьюмера, компонент полностью controlled).
- Правая колонка: остальные 3 пути кнопками (иконка + подпись + шеврон).
- **A · название+город:** `businessName` (обязательное), `city` + автодополнение: `citySuggestions: string[]`, `onCityQuery`, выбор мышью/стрелками/Enter, раскрытие **в потоке** (не поверх контента); свободный ввод города остаётся валидным (строка «Оставим как есть»). Источник списка городов — консьюмер (проп), в канон список не запекать.
- **B · скриншот карточки:** дроп-зона (drag / Ctrl+V / выбор файла), только изображения ≤ 8 МБ, после выбора превью с именем и удалением; `file`, `onFile`, `onRemove`, `fileError`.
- **C · ссылка:** `url`, `onUrlChange`, `detected` (yandex_maps|2gis|telegram|avito|vk|null — по скриншоту подпись пути: «Карты, Telegram, Avito, ВКонтакте»; финальный список подтвердить с founder), `urlError`; зелёная подсказка «источник узнан», красное предупреждение для не-ссылки, **валидация по blur, не во время набора**; блок CTA с причиной.
- **D · фото работ/прайса:** до **5** фото (не 5..60 как в старом флоу), превью с удалением, счётчик «N из 5»; `files`, `onFiles`, `onRemove`.
- Ошибки-возвраты: `notFound` («✗A» — показать искомый запрос, ввод сохранён, рядом другие пути) и `recognizeFailed` («✗B» — объяснение + предложение ввести название руками) — как состояния этого шага (или отдельные лёгкие компоненты — на ваш вкус, но состояния обязаны быть).

### 3.3 `In2_StepRecognize` — шаг 2.5 (только путь «скриншот»)

- Спиннер «Читаем название и город со скриншота…», кнопка «Отменить» (`onCancel`, возврат без потери ввода). Таймаут/успех решает консьюмер — компонент чисто презентационный.

### 3.4 `In2_StepConfirmCard` — шаг 03 «Какая карточка ваша?»

- `candidates: { id, name, address, rating, reviewCount, source: 'yandex_maps'|'2gis' }[]` — выбор карточек-плиток (по скриншоту: карточки в ряд с mono-бейджем источника ЯНДЕКС КАРТЫ / 2ГИС, адрес, «★ 4.9 · 137 отзывов»); `selectedId`, `onSelect`, CTA **«Да, собирать →»** (`onConfirm`), текст-ссылка **«Ничего из этого — искать ещё»** (`onReject` → возврат на источник с подсказкой — состояние 03✗).

### 3.5 `In2_StepBooking` — шаг 04 «Запись»

- Чипы: Dikidi (по умолчанию) · YClients · По телефону и в мессенджерах · Пока никак; `platform`, `onPlatformChange`.
- Dikidi/YClients: поле ссылки **необязательное** (пусто → «возьмём из карточки»); мусор вместо ссылки → красная подсказка + блок CTA с причиной. Телефон: необязательное поле с маской `+7 (___) ___-__-__`, неполный номер блокирует «Дальше». «Пока никак» — поясняющая строка про кнопку «Написать» в мессенджер.
- Правая колонка «Как это будет на сайте» — 3 пункта.

### 3.6 `In2_StepContacts` — шаг 05 «Контакты»

- Чипы каналов — по скриншоту **пять**: **Telegram · MAX · WhatsApp · Email · Телефон/SMS** (текст спеки перечисляет четыре без MAX — верен скриншот; WhatsApp — новый канал) + одно поле контакта; **значение хранится отдельно на каждый канал** (переключение не стирает) — стейт у консьюмера, компонент controlled.
- Маска телефона по мере ввода; email-валидация по blur; причина блокировки под CTA («Осталось: контакт и согласие»). Согласие — со ссылками на документы (href пропсами). Капча невидимая — чекбокса нет.
- CTA «Получить черновик» (`onSubmit`, `submitting`).

### 3.7 `In2_StepDone` — финал «Собираем черновик»

- По скриншоту: чек-иконка, заголовок «Собираем ваш черновик», строка «Пришлём в Telegram на @… примерно через 2 часа» + инлайн-ссылка **«Изменить»** (`onEditContact` → возврат на 05), блок «ЧТО ДАЛЬШЕ» (01/02/03 mono-нумерация), CTA **«Понятно»** (`onClose`).
- Бейдж на лендинге «Собираем: [бренд] · ~2 ч» — уже существует на консьюмере, в пакет не входит.

## 4. Реестр состояний для canvas (артборды приёмки)

Все состояния из спеки, каждое — артборд: `01`, `01·з (скелет)`, `A`, `A+ (автодополнение)`, `A✎ (город свободным вводом)`, `B (пустая дроп-зона)`, `B+ (файл выбран)`, `C✓`, `C✗`, `D`, `2.5`, `03`, `03✗`, `04`, `04✗`, `05`, `05·тел`, `05·email (ошибка)`, `✓`, `✗A`, `✗B`, `⚠ (сбой отправки)`, `✕? (подтверждение закрытия)`, `↺ (черновик восстановлен)`.

**Mobile 390 — обязательная часть поставки.** Спецификация — десктопная (окно 960, две колонки); основной трафик мобильный. Нужна мобильная раскладка минимум для: 01 (пример фуллскрин + табы), 02 (пути стопкой: как складываются «правые» альтернативы), 03, 04, 05, ✓ — с выживанием на 375. Если мобильная версия поедет вторым пакетом — явно сказать в CHANGELOG, но прод не включим до её получения.

## 5. Аналитические хуки `data-*`

События вешает консьюмер по таблице аналитики из спеки (`form_open … draft_reset`, dataLayer-совместимо). Канону — проставить стабильные атрибуты:

- `data-intake-step="example|source|recognize|confirm|booking|contacts|done"` на корне каждого шага;
- `data-niche-id` на табах ниш; `data-source-path="name|screenshot|link|photo"` на переключателях путей; `data-candidate-idx` на карточках кандидатов; `data-booking-platform` на чипах записи; `data-contact-channel` на чипах каналов;
- `data-cta` на primary-CTA каждого шага (`claim-example`, `source-next`, `card-confirm`, `booking-next`, `submit`).

Менять атрибуты после релиза — только согласованно (на них цели Метрики и pixel-diff).
