## Recurring deploys (the happy path)

```bash
ssh deploy@135.106.137.30
sudo -u deploy /opt/vitrina/infra/scripts/deploy.sh --staging
```

`deploy.sh` is idempotent and:

1. `git fetch && git pull --ff-only origin main` (rejects non-FF)
2. `docker compose build --pull` (Buildkit, IPv4-forced apt — see troubleshooting doc)
3. Brings postgres + redis up, waits for healthy
4. `alembic upgrade head` inside the api container
5. `docker compose up -d` for the rest of the services
6. `curl https://samosite.online/healthz` until 200 (90 s budget)
7. On failure of (4) or (6): `git checkout` rollback SHA, rebuild, `up -d`
8. Writes `state-snapshot-YYYYMMDD-HHMM.json` for diffing

Logs: `/var/log/vitrina/deploy-YYYYMMDD.log` (also streamed to stderr).

### Useful flags

| Flag | When |
|---|---|
| `--no-pull` | Redeploying the same SHA (e.g. after fixing `.env`) |
| `--no-build` | Just re-applying compose state, no image changes |

## First-time bring-up (what was done on 2026-05-20)

This section is reference-only; everything below now lives in scripts
or is one-shot infra. Don't re-run on a healthy host.

### Phase 1 — DNS

Confirmed the four Selectel nameservers serve A records for apex, www,
and a wildcard:

```bash
dig +short samosite.online                 # → 135.106.137.30
dig +short www.samosite.online             # → 135.106.137.30
dig +short test.samosite.online            # → 135.106.137.30 (wildcard)
dig +short NS samosite.online              # → {a,b,c,d}.ns.selectel.ru
```

### Phase 2 — secrets discovery

`/opt/vitrina/.env` is the canonical compose env-file. Crypto secrets
live in `/opt/vitrina/.env.secrets` (`chmod 600`, owned by `deploy`).
P0 secrets (FERNET_KEYS, JOB_HMAC_KEY, BACKUP_PASSPHRASE,
SESSION_SECRET_KEY, all postgres role passwords) are generated by
`infra/scripts/bootstrap-vps.sh` and merged into `.env` at deploy time.

P1 secrets still placeholder-marked (acceptable per ADR-0008 graceful
no-op): `YANDEXGPT_API_KEY`, `YANDEX_GEOSEARCH_API_KEY`,
`YANDEX_SMARTCAPTCHA_*`, `TG_BOT_TOKEN`, `S3_*`, `YOOKASSA_*`,
`SELECTEL_DNS_API_TOKEN`, `ACME_EMAIL`. Each consuming module reads its
own `FEATURE_*_ENABLED` flag; absent keys → silent no-op, not crash.

### Phase 3 — DB migrations + roles

```bash
cd /opt/vitrina
docker compose --env-file .env \
    -f infra/docker-compose.yml -f infra/docker-compose.staging.yml \
    up -d postgres
# `infra/postgres/init/01_roles.sh` runs once on first boot from the
# POSTGRES_* env vars and creates the four roles.
docker compose ... exec postgres psql -U vitrina_migrator -d vitrina \
    -c "\du" | grep vitrina_
# vitrina_migrator | Superuser | …
# vitrina_app      | DML only  | …
# vitrina_readonly | SELECT    | …
# vitrina_audit_writer | INSERT on admin_actions | …

docker compose ... run --rm api alembic upgrade head
# 0001_init -> 0006 applied in order; idempotent on re-run
```

### Phase 4 — admin seed

```bash
docker compose ... run --rm api python -m app.cli.seed_admin
# interactive: username, password (bcrypt cost=12), TOTP enrollment,
# 8 backup codes. Output → 1Password "Vitrina admin (founder)".
```

### Phase 5 — full stack

```bash
docker compose --env-file .env \
    -f infra/docker-compose.yml -f infra/docker-compose.staging.yml \
    up -d
```

9 containers (postgres, redis, api, parser-worker, content-worker,
sync-worker, tg-bot, landing, caddy). Workers go on `parser_net` (no
postgres route — verified with `nc -w2 postgres 5432` returning
connection refused).

### Phase 6 — Caddy + TLS

Apex + www succeed via Let's Encrypt **tls-alpn-01** on port 443. The
wildcard host falls back to **tls internal** (self-signed) until
`SELECTEL_DNS_API_TOKEN` is provided and the `acme` sidecar runs
DNS-01 issuance — see [`infra/acme/`](../../infra/acme/) and
`infra/Caddyfile` for the canonical wildcard block.

### Phase 7 — smoke tests

```bash
# from operator laptop:
curl -I  http://samosite.online              # → 308 to https
curl -I  https://samosite.online             # → 200, HSTS present
curl     https://samosite.online/healthz     # → ok
curl     https://samosite.online/readyz      # → ok
curl -I  https://samosite.online/admin/login # → 200, Самосайт chrome
curl -kI https://test.samosite.online        # → 404 self-signed (expected)

# brand visibility:
curl -s https://samosite.online | grep -c 'Самосайт'  # > 0
```

### Phase 8 — backups

```bash
sudo install -d -o deploy -g deploy /var/log/vitrina
sudo install -o deploy -g deploy /dev/stdin /etc/cron.d/vitrina-backup <<'EOF'
SHELL=/bin/bash
PATH=/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin
0 3 * * * deploy /opt/vitrina/infra/scripts/backup-db.sh > /var/log/vitrina/backup-cron.log 2>&1
EOF
sudo install -o root /dev/stdin /etc/logrotate.d/vitrina <<'EOF'
/var/log/vitrina/*.log {
    daily
    rotate 14
    compress
    delaycompress
    missingok
    notifempty
    create 0644 deploy deploy
    sharedscripts
}
EOF
# kick off first backup immediately to verify the pipe end-to-end:
sudo -u deploy /opt/vitrina/infra/scripts/backup-db.sh
ls -la /opt/vitrina/backups/   # → db-YYYYMMDD-HHMM.sql.gpg
```

## Common ops tasks

### Update without code changes (e.g. rotated a secret in `.env`)

```bash
sudo -u deploy /opt/vitrina/infra/scripts/deploy.sh --staging --no-pull --no-build
```

### Force rebuild (after editing a Dockerfile locally and pushing)

```bash
sudo -u deploy /opt/vitrina/infra/scripts/deploy.sh --staging
# the --pull flag inside the script picks up new base images too
```

### Rollback to a specific commit

```bash
ssh deploy@135.106.137.30
cd /opt/vitrina
git fetch origin
git checkout <commit-sha>
sudo -u deploy /opt/vitrina/infra/scripts/deploy.sh --staging --no-pull
```

### View running stack

```bash
docker compose --env-file .env \
    -f infra/docker-compose.yml -f infra/docker-compose.staging.yml \
    ps
```

### Tail a service log

```bash
docker compose ... logs -f --tail=200 api          # API access + errors
docker compose ... logs -f --tail=200 caddy        # TLS + reverse proxy
docker compose ... logs -f --tail=200 content-worker  # YGPT calls
```

### Restore from backup

```bash
# 1. Pick a backup
ls -1 /opt/vitrina/backups/
BACKUP=/opt/vitrina/backups/db-20260520-2150.sql.gpg

# 2. Decrypt
BACKUP_PASS=$(grep '^BACKUP_PASSPHRASE=' /opt/vitrina/.env.secrets | cut -d= -f2-)
gpg --batch --passphrase "$BACKUP_PASS" --decrypt "$BACKUP" > /tmp/restore.sql

# 3. Stop dependent services (keep postgres up)
cd /opt/vitrina
docker compose ... stop api parser-worker content-worker sync-worker tg-bot landing caddy

# 4. Drop + recreate DB
PG_PW=$(grep '^POSTGRES_PASSWORD=' /opt/vitrina/.env.secrets | cut -d= -f2-)
PGPASSWORD="$PG_PW" docker compose ... exec -T postgres \
    psql -U vitrina_migrator -d postgres -c "DROP DATABASE vitrina;"
PGPASSWORD="$PG_PW" docker compose ... exec -T postgres \
    psql -U vitrina_migrator -d postgres -c "CREATE DATABASE vitrina OWNER vitrina_migrator;"

# 5. Restore
PGPASSWORD="$PG_PW" docker compose ... exec -T postgres \
    psql -U vitrina_migrator -d vitrina < /tmp/restore.sql
shred -u /tmp/restore.sql

# 6. Bring everything back
sudo -u deploy /opt/vitrina/infra/scripts/deploy.sh --staging --no-pull --no-build
```

Quarterly restore drill (per SECURITY.md §11): do steps 1–4 against a
throwaway DB name (`vitrina_restore_test`), verify table counts, drop
it. Don't actually replace prod.

## Deferred / known limitations

- **Wildcard TLS is `tls internal`.** Subdomains will show "Not Secure"
  in browsers until `SELECTEL_DNS_API_TOKEN` is wired through
  `infra/acme/`. Plan: set token in `.env`, run `docker compose run
  --rm acme /scripts/issue.sh`, swap `Caddyfile.staging` wildcard block
  to use volume cert, `docker compose up -d caddy`. Verify externally
  with `curl -vI https://test.samosite.online` showing `LE E7` issuer.
- **`ACME_EMAIL` is a placeholder** (`founder@samosite.online`). Let's
  Encrypt sends expiry warnings here; replace with a monitored mailbox
  before public launch.
- **P1 product features are no-op** until their keys are provided:
  YandexGPT generation, SmartCaptcha, TG bot, S3 uploads, ЮKassa.
  Lead-intake form on customer sites still works (encrypted persistence
  works; SmartCaptcha gate becomes effective once key is set).
- **No off-host backup yet.** Backups live on the same VPS. Migrate to
  Selectel Cold Storage (separate region) before public launch — see
  SECURITY.md §11 pre-launch checklist.
