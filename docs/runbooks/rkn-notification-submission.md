# Runbook — Уведомление РКН об обработке ПДн (T6.4)

> **Status:** документ — действие выполняет фаундер до запуска формы заявок на лендинге. Не код. Этот runbook — реестр готовности и точная формулировка полей для подачи через `pd.rkn.gov.ru`.

**Кто исполняет:** Founder (ИП Vitrina).
**Срок:** до публичного запуска — иначе ст. 18.3 ФЗ-152 + ст. 13.11 КоАП РФ (штрафы до 75 000 ₽ за каждый случай).
**Стоимость:** 0 ₽. Подача и рассмотрение бесплатные. Срок рассмотрения регулятора — до 30 календарных дней.

- [ ] **ИП зарегистрировано.** ОГРНИП + ИНН известны.

- [ ] **Сверен `consents.policy_version` в DB-ledger** с версией политики на сайте (`backend/app/core/consent/ledger.py::CURRENT_POLICY_VERSION`).

## 3. Шаги подачи

1. Залогиниться на `https://pd.rkn.gov.ru` через ЭЦП.
2. «Подать уведомление» → «Об обработке (о намерении осуществлять обработку) персональных данных».
3. Заполнить разделы 1–14 по таблице из §2.
4. Подписать ЭЦП и отправить.
5. Сохранить **PDF-квитанцию** в `docs/legal/rkn-notification-submitted-YYYY-MM-DD.pdf` (внутри репо, не в 1Password — это нечувствительный документ).

## 5. Изменения после подачи

ФЗ-152 ст. 22 ч. 7: при изменении сведений из уведомления подавать «Уведомление об изменении» в течение 10 рабочих дней. Триггеры:
- Новый источник ПДн (новый адаптер парсера → новые категории субъектов / способы).
- Новый провайдер инфраструктуры.
- Бампанули `consents.policy_version` (новый юридический текст).
