# @samosite/canon (vendored UI source of truth)

> Reads root `CLAUDE.md` first. Applies when working **on** canon, not when consuming it from landing.

## What this package is

`@samosite/canon` is Claude Design's React source vendored as a workspace `file:` dep at `packages/canon/`. It IS the UI source of truth — every screen, primitive, and token. Prod imports from it directly, making **drift mathematically impossible** as long as the rules below hold.

Produced by Claude Design per `docs/handoff/CANON_PACKAGE_TZ.md` (Variant C).

## What lives here

- **19 screens** — landing × 13 + intake × 6 + customer × 1 + admin × 10 + admin-demo × 1 (mapped in `docs/handoff/SCREEN_INDEX.md`)
- **12 primitives** — `SectionTitle`, `SectionSub`, `Btn`, `Mono`, `Logo`, `BrandMark`, icons, `FeatureCard`, etc.
- **Design tokens** — `VT`, `BRAND`, flat `tokens` (colors, fonts, shadows, radii)
- **`<CanonStyles />` provider** — hover-lift, focus rings, vt-pulse keyframes, smooth-scroll
- **Tailwind preset** — `@samosite/canon/tailwind-preset`

## Hard rules

### NEVER

- NEVER edit `packages/canon/src/*` directly. **This is vendored.** Changes come from Claude Design as full-package replacements.
- NEVER hand-write into `packages/canon/dist/` — generated by `npm run build` from `src/`.
- NEVER bump `packages/canon/package.json` version unilaterally — semver is tracked in Claude Design's workflow.

### ALWAYS

- ALWAYS round-trip changes through Claude Design: file an issue describing the gap → wait for design to ship a new package version → run the refresh procedure in `landing/CLAUDE.md` § "Canon refresh procedure".
- ALWAYS commit `packages/canon/dist/` — consumers depend on built output. Pre-commit hooks exempt `dist/` from whitespace/EOF normalization.

## When to file an issue with design (instead of patching here)

- A prod screen needs a state/variant canon doesn't export
- A token is missing (color, spacing, radius)
- A primitive is needed but doesn't exist
- Existing canon component has a bug (incorrect padding, wrong color)
- Pixel-diff against canon fails AND the prod implementation is verifiably correct

In all these cases: **do not patch `src/`**. File the issue, log the gap in `CHANGELOG.md` "Known gaps", work around on consumer side with `@todo canon-gap-NNN` comment.

## What CAN be edited here (rare)

- `packages/canon/CHANGELOG.md` — log known gaps consumers must be aware of, document recent releases
- `packages/canon/README.md` — consumer-facing docs (entry points, fonts setup, Tailwind preset)
- `packages/canon/CLAUDE.md` — this file

Edits to anything else require an ADR or design-team coordination.
